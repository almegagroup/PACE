## Frontend Deploy Neutrality

### Status
- **DONE**
- **FROZEN**

### Purpose
Ensure the frontend is:
- Browser-agnostic
- Electron / PWA / Mobile WebView safe
- Free from browser-only assumptions

### Core Rules
1. Frontend must not assume browser globals exist.
2. No direct usage of `window`, `document`, or `navigator` without guards.
3. No client-side session or cookie manipulation.
4. Navigation must be router-driven, not via `window.location`.

### Runtime Guard
A centralized runtime guard is defined:

**File:** `src/utils/runtime.js`

```js
export const isBrowser =
  typeof window !== 'undefined' &&
  typeof document !== 'undefined';

export const isNavigator =
  typeof navigator !== 'undefined';
All browser-specific logic must reference this guard.

Absolute Bans
The following APIs are permanently forbidden:

window.alert

window.confirm

window.prompt

Direct cookie access

navigator.userAgent logic branching

Audit Proof
Static scans confirm absence of:

window.location

document.cookie

localStorage / sessionStorage

Browser-only assumptions

Invariant
Frontend logic must execute identically across browser, Electron, and mobile runtimes.
